## 1. Goal

Prepare the data/config foundation so subsequent v2 features (identity/bind, roster sync, Discord role sync) can plug in **without changing v1 behavior** of `courses`, `users`, and the mark cache.

## 2. Scope

**In scope** (matches issue *Phß║Īm vi* + acceptance criteria):

- Extend `configs.Config` and env loading for Discord / SMTP / OTP / roster / new collections.
- Add domain model + repository for `students`, `bindings`, `verifications`, `discord_mappings`.
- Add idempotent index bootstrap:
  - `students.email` ŌĆö unique
  - `bindings(platform, platform_user_id)` ŌĆö unique
  - `bindings(platform, mssv)` ŌĆö unique
  - `bindings.mssv` ŌĆö non-unique
  - `verifications.expiry` ŌĆö TTL (`expireAfterSeconds: 0`, stored as BSON Date)
- Preserve compatibility with `courses`, `users`, and mark collections.

**Out of scope** (deferred to later tasks ŌĆö keeps v1 untouched):

- `user.Role` extension and the `users` migration (unset `is_teacher`).
- Port interfaces `domain/discord.Bot`, `domain/email.Sender`.

- `cmd/discord` service binary.
- Any use-case logic (`identity`, `rostersync`, `classsync`) and any rewiring of `api`/`fetcher`/`tele` with the new repos.

## 3. Design

### 3.1 Config (`internal/configs/config.go`) ŌĆö additive

Add v2 fields per `config-env.md` ┬¦7:

```go
// v2 ŌĆö Mongo collections
DbSettingsStudents, DbSettingsBindings, DbSettingsVerifications, DbSettingsDiscordMappings string
// v2 ŌĆö Discord
DiscordToken, DiscordGuildId string
DiscordAdminIds             []string
// v2 ŌĆö Email / OTP
SmtpHost, SmtpUsername, SmtpPassword, SmtpFrom string
SmtpPort                                      int
OtpLen                                        int
OtpTtl                                        time.Duration
// v2 ŌĆö Sync
RosterCsvUrl                   string
RosterSyncInterval, RoleSyncInterval time.Duration
```

New helpers (reusing existing `loadEnv` style):

- `loadEnvInt(key string, defaultValue int) int` ŌĆö `strconv.Atoi`, falls back to default.
- `loadEnvDuration(key string, defaultValue time.Duration) time.Duration` ŌĆö `time.ParseDuration`, falls back to default.

JSON-array parsing reuses existing `loadEnvJsonSlice` (e.g. `loadEnvJsonSlice[string]("DISCORD_ADMIN_IDS", []string{})`).

Defaults match `config-env.md`: `DB_SETTINGS_STUDENTS=students`, `..._BINDINGS=bindings`, `..._VERIFICATIONS=verifications`, `..._DISCORD_MAPPINGS=discord_mappings`, `SMTP_PORT=587`, `OTP_LEN=6`, `OTP_TTL=5m`, `ROSTER_SYNC_INTERVAL=24h`, `ROLE_SYNC_INTERVAL=30m`. Secrets (`DISCORD_TOKEN`, `SMTP_*`, `DISCORD_GUILD_ID`, `ROSTER_CSV_URL`) default to `""` and are supplied via SOPS `enc.env` at runtime ŌĆö no change to secret handling.

> **Note (minor extension):** `DB_SETTINGS_DISCORD_MAPPINGS` is not listed in `config-env.md` ┬¦1, but is added for consistency with the other configurable collection names. It is trivially removable if literal doc adherence is preferred.

All v1 fields and defaults remain byte-for-byte unchanged.

### 3.2 Domain packages

Each new package mirrors the existing `course`/`user` layout: `model.go` + `repo.go`, with a package-level `ErrNotFound`. Repository methods take **no `context.Context`** argument ŌĆö each implementation derives its own `context.WithTimeout(background, timeout)`, matching `course_repo.go`/`user_repo.go` exactly.

#### `internal/domain/student`
```go
type Model struct {
    MSSV  string `json:"mssv"  bson:"_id"`
    Name  string `json:"name"  bson:"name"`
    Email string `json:"email" bson:"email"`
}

type Repository interface {
    Upsert(m Model) error
    FindByEmail(email string) (Model, error) // bind flow: email -> MSSV
    FindByMSSV(mssv string) (Model, error)
}
```

#### `internal/domain/binding`
```go
type Model struct {
    Platform       string `json:"platform"         bson:"platform"`
    PlatformUserID string `json:"platform_user_id" bson:"platform_user_id"`
    MSSV           string `json:"mssv"             bson:"mssv"`
    Verified       bool   `json:"verified"         bson:"verified"`
    BoundAt        int64  `json:"bound_at"         bson:"bound_at"`
}

type Repository interface {
    Upsert(m Model) error
    FindByPlatformUser(platform, platformUserID string) (Model, error)
    FindByPlatformMSSV(platform, mssv string) (Model, error)
    FindByMSSV(mssv string) ([]Model, error)
}
```

`_id` is auto-generated by Mongo; the two unique compound indexes serve as natural keys. `Upsert` filters on `(platform, platform_user_id)`.

#### `internal/domain/verification`
```go
type Model struct {
    PlatformUserID string    `json:"platform_user_id" bson:"_id"`
    Email          string    `json:"email"            bson:"email"`
    OTP            string    `json:"otp"              bson:"otp"`
    Expiry         time.Time `json:"expiry"           bson:"expiry"`
}

type Repository interface {
    Upsert(m Model) error   // _id = PlatformUserID; overwrite per OTP attempt
    Find(platformUserID string) (Model, error)
    // Deletion is handled by the TTL index; no explicit Delete in foundation.
}
```

`Expiry` is `time.Time`, which the driver serializes as a BSON Date ŌĆö required for the TTL index.

#### `internal/domain/discordmapping`
```go
type Model struct {
    CourseId         string `json:"course_id"          bson:"_id"`
    DiscordRoleId    string `json:"discord_role_id"    bson:"discord_role_id"`
    DiscordChannelId string `json:"discord_channel_id" bson:"discord_channel_id"`
}

type Repository interface {
    Upsert(m Model) error
    Find(courseId string) (Model, error)
    Remove(courseId string) error
}
```

### 3.3 Mongo repositories (`internal/infra/mongo/*_repo.go`)

Each follows the existing `NewXxxRepo(client *Client, config *configs.Config) <domain>.Repository` constructor (see `user_repo.go`), holding `db *mongo.Database`, `collection *mongo.Collection`, `timeout time.Duration`.

- `Upsert` uses `collection.UpdateOne`/`UpdateByID` with `options.Update().SetUpsert(true)`:
  - `student` / `discordmapping` / `verification`: filter on `_id` (MSSV / CourseId / PlatformUserID).
  - `binding`: filter on `{platform, platform_user_id}`, `$set` the model.
- `Find*` map `mongo.ErrNoDocuments` ŌåÆ the domain `ErrNotFound` (as `user_repo.go` does).

### 3.4 Index bootstrap (`internal/infra/mongo/indexes.go`)

```go
func EnsureIndexes(client *Client, config *configs.Config) error
```

Creates indexes via `collection.Indexes().CreateMany(ctx, models)` with **explicit names** so re-runs are idempotent no-ops (re-creating an identical existing index succeeds silently):

| Collection | Index name | Keys | Options |
|---|---|---|---|
| `students` | `uniq_email` | `{email: 1}` | unique |
| `bindings` | `uniq_platform_user` | `{platform: 1, platform_user_id: 1}` | unique |
| `bindings` | `uniq_platform_mssv` | `{platform: 1, mssv: 1}` | unique |
| `bindings` | `idx_mssv` | `{mssv: 1}` | ŌĆö |
| `verifications` | `ttl_expiry` | `{expiry: 1}` | `expireAfterSeconds: 0` |

`discord_mappings` has only the implicit `_id` index ŌĆö nothing to create. **`courses`/`users`/marks are never touched.**

### 3.5 Wiring (minimal ŌĆö no Wire regeneration)

The bootstrap is invoked directly from each existing `main.go` (api, fetcher, tele) immediately after `InitializeApp`, the same way `infra.InitDotenv()` is called today:

```go
app, err := InitializeApp()
if err != nil { log.Fatal()... }
if err := mongo.EnsureIndexes(app.MongoClient, app.Config); err != nil {
    log.Fatal().Err(err).Msg("ensure indexes failed")
}
```

Rationale for running in all three services: idempotent, affects only new collections, and self-bootstraps whichever service is deployed. Startup cost is negligible (a handful of `createIndex` no-ops).

The new repo constructors are added to package `mongo` but **not** registered in any `wire.Build` ŌĆö they have no consumer yet (later use-case tasks will wire them). No `wire.go` / `wire_gen.go` changes, so v1 service composition is byte-for-byte identical apart from the added idempotent bootstrap call.

## 6. Addendum ŌĆö OTP anti-abuse (added after review + owner direction)

Owner confirmed the **15-min resend cooldown is a must**, **OTP validity = 15m**, and directed an **anti-abuse design** (reviewer Codex raised the resend-cooldown gap on PR #8). This addendum supersedes the earlier OTP defaults in ┬¦3.1 (was `OTP_TTL=5m`).

**Policy (codified in `config-env.md` ┬¦4.1 and `SRS-v2.md` ┬¦6.1.1):**

- `OTP_TTL` default `5m ŌåÆ 15m`; this is **also the resend-cooldown window**. Cooldown is decided by `expiry > now` on the existing `verification` record ŌĆö **not** by record existence (Mongo TTL deletion is async, so an expired doc can linger past `expiry`; the TTL index is eventual cleanup only, not the enforcement mechanism).
- New config `OTP_MAX_ATTEMPTS` (default `5`) ŌĆö brute-force cap.
- Brute-force hole avoided: on reaching the cap, the OTP is **invalidated but the record is kept** (`expiry > now` still holds ŌåÆ resend stays blocked) ŌĆö so an attacker can't immediately re-request to get fresh attempts.
- Per-`email` cooldown (Sybil defense): **unique** index `verifications.email` enforces "at most one live OTP per email" **atomically** at the DB level ŌĆö two concurrent binds for the same email but different `platformUserID` cannot both create a record.

**Foundation changes (this PR):**

- `config`: `OTP_TTL=15m`; `OtpMaxAttempts int` (default 5).
- `verification.Model`: `Attempts int bson:"attempts"`; `Upsert` resets it to `0` on each new OTP.
- `verification.Repository`: `IncrementAttempts(platformUserID) (int, error)` ŌĆö atomic `$inc`, returns new count; `FindByEmail(email) ([]Model, error)`.
- index `verifications.uniq_email` (**unique**) ŌĆö atomic per-email cooldown (Sybil defense).

**Deferred to `usecases/identity`** (enforcement): check cooldown before send, `$inc` on wrong OTP, invalidate at `OtpMaxAttempts` (keep record), per-email check.
